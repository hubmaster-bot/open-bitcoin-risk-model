# ADR-017: Maintain documentation as a first-class project layer

## Status

Accepted

## Decision

OBRM documentation is organised into:

- User Guide
- Research Handbook
- Developer Guide
- Architecture Decision Records
- Changelog
- In-app Documentation Hub

## Rationale

The code explains how OBRM works.

The documentation must also explain:

- why it works this way,
- how to interpret it,
- where it is limited,
- and how to extend it safely.

Documentation must evolve alongside the software rather than being deferred until v1.0.
